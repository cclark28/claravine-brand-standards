Design System · Typography

Typography

Type defines the voice of the brand. We use two complementary typeface families: Poppins for headings and UI, Roboto for body text and longer-form content.

Type Scale

All font sizes and weights follow this consistent scale. Use these values in CSS custom properties (e.g., var(--text-lg)) for easy maintenance.

Token Size Usage
--text-5xl 54px Hero titles, major headings
--text-4xl 42px Section titles, page headings
--text-3xl 32px Subsection headings
--text-2xl 24px Card titles, emphasis
--text-xl 20px Subheadings
--text-lg 18px Body prominent
--text-base 16px Body text (primary)
--text-sm 14px Labels, captions
--text-xs 12px UI text, badges, metadata
--text-2xs 11px Small labels, footnotes

Primary Typeface — Poppins

Poppins is a modern, geometric sans-serif. Used for all headings, UI, and emphasis text. Weights: 300 (Light), 400 (Regular), 500 (Medium), 600 (Semibold), 700 (Bold).

Shadow usage rationale

Use is-shadow on elevated or focused components: floating action buttons, sticky CTAs, modal confirm actions, and inline form submit buttons. Do not use shadow on inline text actions, ghost buttons, or buttons inside cards that already have elevation. Shadow communicates "this button can act right now" — reserve it for primary and dark variants only.

Clear Space & Minimum Size

Always maintain clear space equal to the cap-height of the wordmark around all edges. Minimum rendered width: 120px for full logo, 24px for step mark only.

Logo Do's and Don'ts

Follow these guidelines to maintain brand integrity across all media.

✓ Do
Use full color logo on white or light neutral backgrounds.
✓ Do
Use white wordmark + green step mark on brand dark background.
✓ Do
Scale uniformly. Keep wordmark and step mark proportional.
✗ Don't
Do not stretch, compress, or distort the logo in any direction.
✗ Don't
Do not place the logo on backgrounds with insufficient contrast.
✗ Don't
Do not recolor the wordmark or step mark in non-brand colors.
✗ Don't
Do not add drop shadows, gradients, or visual effects to the logo.
✗ Don't
Do not rotate, tilt, or apply any transformations to the logo.

Brand Guidelines · Voice

Voice & Tone

Claravine's voice is clear, confident, and human. We speak to marketers and agencies as peers, not as a vendor. Our tone adjusts to context but never loses the brand character.

Voice Attributes

Clear

Cut through jargon. Use simple words. Explain things like you're talking to a smart colleague.

Confident

Stand behind what we say. Use active voice. Be direct and bold.

Human

Write like a person, not a machine. Show personality. Be relatable.

Tone Across Contexts

Helpful (Support)

Patient. Solution-focused. Acknowledge the problem before the fix.

Educational (Docs)

Instructive. Show examples. Break down concepts.

Energetic (Marketing)

Optimistic. Inspiring. Focus on possibilities.

Do's and Don'ts

DO
  • Use active voice: "We solve data chaos"
  • Be conversational: "Let's get started"
  • Show respect: "Your data deserves clarity"
  • Use contractions: "We're here to help"
  • Be specific: "46% of marketers report..."
DON'T
  • Use corporate jargon: avoid "synergy," "paradigm shift"
  • Be passive: "mistakes were made"
  • Sound like a bot: "click the button to proceed"
  • Use ALL CAPS unless it's a proper noun
  • Condescend or over-explain
Design System · Motion

Motion & Animation

Motion brings the brand to life. Every animation serves a purpose: it guides attention, communicates state, or provides feedback. Motion should feel natural, not jarring.

Easing Functions

Use these easing functions across all animations for consistency.

ease-in-out

cubic-bezier(0.4, 0, 0.2, 1)

Use for most UI transitions and state changes.

ease-out

cubic-bezier(0, 0, 0.2, 1)

Use for entrances and appears.

ease-in

cubic-bezier(0.4, 0, 1, 1)

Use for exits and dismissals.

Animation Duration

Duration should feel immediate, not instant or slow. Follow these guidelines:

Micro-interactions (loading spinners, icon state changes)

140ms – 180ms

UI transitions (fade, slide, drawer)

220ms – 300ms

Page transitions (hero animations, background loops)

1200ms – 2000ms+

Motion Demo Samples

Below are visual examples of standard motion patterns.

Reduced Motion & Accessibility

Always respect prefers-reduced-motion. Users with vestibular disorders may experience nausea with certain animations. Disable hero animations and reduce overall duration for these users.

Architecture · Governance

Living System

This guide is not a static document — it is a living system designed to stay current, move with the product, and transfer cleanly to any infrastructure Claravine requires.

public

Accessible

Published continuously to a public URL via GitHub Pages. No login, no VPN, no gated PDF. Any designer, developer, or partner can reference the current standard at any time from any device.

autorenew

Adaptable

Built as a single self-contained HTML file with CSS custom properties as design tokens. Updating a color, typeface, or spacing value in one place cascades across the entire guide instantly — no build step, no dependencies to manage.

move_up

Transferable

The guide is a portable, dependency-light file that can be copied to any web server, embedded in an intranet, integrated into a WordPress site, or served from Claravine's own domain — without modification to the source.

Migration to Internal Servers

When Claravine is ready to host this guide on internal infrastructure — as a subdomain, within claravine.com, or on a private intranet — the migration requires no code changes. The guide is a single portable file with no server-side dependencies.

Step 01

Export from GitHub

Clone or download the repo. The entire guide is index.html plus assets/logos/.

Step 02

Upload to Internal Server

Place the file on your server. If using HTTPS (recommended), the guide works immediately.

Step 03

Point Domain

Configure DNS or a subdomain to point to the server where the file lives.
