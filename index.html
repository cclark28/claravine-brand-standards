⚠ Site Inconsistencies Detected

The following were found on claravine.com that deviate from this brand guide. These should be corrected in the WordPress theme.

Off-Brand Color H2 headings on homepage use rgb(29, 144, 255) — a bright blue not in the approved color palette. H2 headings should use #263F48 Brand Dark Navy or #5DD39E Claravine Green for emphasis. Update in WordPress theme's h2 { color } CSS.
Off-Brand Color Purple background found on homepage — approximately rgb(165, 124, 235). Purple is not in the Claravine color palette. This element should use Brand Dark Navy (#263F48), Claravine Green (#5DD39E), or Sky Blue (#93DCFF) instead. Locate in the WordPress block editor and replace the background color.
Token Mismatch Body text color on live site is #30505A, slightly lighter than the Design Token Dark Navy (#263F48). Use #30505A for body/paragraph text and reserve #263F48 for headings, borders, and UI chrome. Both are now documented in the design tokens above.
WordPress Implementation Note: The site runs a custom Tailwind-based WordPress theme (wp-theme-tailwind-tddtheme). Button styles live in the theme's CSS, not in the block editor. All transition durations and border-radius values above should be treated as the source of truth for the theme stylesheet.

Design System · Logo

Logo Usage

The Claravine logo is minimalist and modern, featuring lowercase typography with a subtle step mark — symbolic of the structured grid associated with data standards. Always use approved files from the asset library.

The Step Mark

The step mark (the small square element appended to the wordmark) represents the disciplined structure of data standards — a reference to the precision of spreadsheet tooling. It must always accompany the wordmark unless using the standalone mark variant.

Clear Space

Maintain a minimum clear space equal to 1× the cap height of the wordmark ("X") on all sides. Never allow other elements, text, or imagery inside this zone.

1X 1X 1X 1X

X = cap-height of the wordmark at the applied size

Logo Do & Don't

check_circle Do
  • Place on a clean, uncluttered background
  • Size proportionately — maintain aspect ratio
  • Use approved color combinations only
  • Ensure sufficient contrast on all backgrounds
  • Use all-white logo on dark backgrounds
  • Maintain required clear space at all times
cancel Do Not
  • Stretch, distort, or rotate the logo
  • Place on busy or high-contrast imagery
  • Alter colors or add drop shadows or effects
  • Rearrange or modify the mark position
  • Use the full-color logo on colored backgrounds
  • Recreate the logo from scratch

Design System · Motion

Motion & Animation

Motion in Claravine's digital presence is purposeful and restrained. Every animation communicates state or guides attention — never decorates for its own sake. Built for WordPress and standard CSS.

Motion Principles

Purposeful

Every animation has a job

Use motion to communicate state change, guide user attention, or confirm an action. Never animate purely for decoration.

Subtle

Never distracting

Keep durations short and displacements small. Motion should feel natural — users shouldn't notice it unless they look for it.

Accessible

Respects prefers-reduced-motion

All animations are wrapped in @media (prefers-reduced-motion: no-preference) so users with vestibular disorders are never affected.

Easing Curves

Use the appropriate easing for the interaction type. Ease Out is the default for most UI transitions.

Ease Out · Default

cubic-bezier(0.0, 0.0, 0.2, 1)

Elements enter the screen. Fast start, gentle finish. Use for dropdowns, modals, and content appearing.

Ease In · Exit

cubic-bezier(0.4, 0.0, 1, 1)

Elements leaving the screen. Slow start, fast finish. Use for elements dismissing or collapsing.

Standard · Repositioning

cubic-bezier(0.4, 0.0, 0.2, 1)

Elements repositioning within the screen. Use for layout changes, reordering, and tab transitions.

Emphasis · Attention

cubic-bezier(0.4, 0.0, 0.6, 1)

High-emphasis moments. Bounce-like feel for important confirmations, success states, or alerts.

Duration Scale

Shorter durations for small UI interactions; longer for page-level transitions.
